# 驗證計畫：先確認有人會用，再大力做

> 來源：兩輪深度研究（含對抗式驗證）。這份是把研究結論落地成「不寫 code 也能做」的驗證步驟。

## 市場與競品（已查證）

**痛點是真的、市場已被驗證**：國外攝影師生意管理 SaaS 是成熟大市場——
- **Pixieset**：完整 suite（選片/交件/合約/報價/預約/收款），數十萬~上百萬攝影師、約 173 員工。
- **HoneyBook / ShootProof / Pic-Time / Sprout Studio**：各有 CRM/合約/金流/選片分工。
- 選片這一步專門工具：**Photo Mechanic（買斷 $139）**。
- AI 自動選片：**Aftershoot（~$10/月）、FilterPixel、Narrative Select**。

**Cullr 的縫**：免費 + 瀏覽器 + 不上傳（隱私）+ AI 輔助——對手多要安裝、要錢、常要上傳。

## 兩個必須面對的風險

1. **願付未知（最大）**：資深攝影師已用免費複製貼上範本 + 土法解決。「免費堪用 → 付月費」很難跨。
   *但 Cullr 本來就免費（作品集/貢獻優先），所以這個風險對「會不會被用」比「會不會付錢」更關鍵。*
2. **「台灣沒有在地對手」只是『沒查到』、非『確定空白』**——研究未窮盡掃 Dcard/PTT/FB，需自己驗。
3. **DIY 替代品已存在**：ECPay 付款連結 + 範本 + Google Drive。產品必須明顯比這個組合好。

## 最小驗證（不寫 code，先做這個）

### 去哪
Dcard 攝影版、PTT photo、婚攝/攝影接案 FB 社團、IG 攝影師私訊。

### 問哪 5 種人
① 婚攝 ② 商攝/產品攝 ③ 個人寫真/家庭 ④ 新手接案 ⑤ 資深接案。

### 問什麼（照順序）
1. 「你現在**選片 / 交件 / 簽約 / 收訂金**整套怎麼跑？」（順便問出他用什麼工具 → 同時驗證風險 2）
2. 「最煩、最常出包的是哪一段？」
3. 「如果有個**免費、照片不上傳、瀏覽器就能用、還會自動標失焦/連拍**的選片工具，你會用嗎？哪裡會讓你不想用？」
4. 「你用過 Photo Mechanic / Aftershoot / Pixieset 嗎？為什麼用 / 不用？」（驗證風險 1、3）

### 綠燈判準
5 個裡有 **3 個以上**在「選片」這段喊痛、且聽到「免費 + 不上傳 + 自動標」會眼睛一亮 → 繼續做。
若大家都「我用 XX 就好」或「還好耶」→ 換族群（補教 / 健身 / 寵物），成本只是幾天。

## 散播（內容導流，非 cold sales）
做出可動 demo 後丟：r/photography、Dcard 攝影版、婚攝/接案 FB 社團。
Hook：**「免費・瀏覽器選片・照片不上傳・不用買 Photo Mechanic」**。

## 仍待釐清（研究的 open questions）
- 願付強度、台灣是否真有在地對手（需窮盡掃社群）
- 最痛的在地層是哪個（合約 / 訂金 / LINE / 發票——目前證據對「選片本身」最直接）
- 主攻婚攝、商攝還是寫真（合約複雜度、訂金、選片量不同）
