# 架構原則：照片永遠本機，後端是選配

## 核心約束（不可妥協）
**使用者的照片永遠不離開瀏覽器。** 這是 Cullr 對抗 Aftershoot / Pixieset 等
「要安裝或要上傳」工具的差異化賣點。任何會把照片送上伺服器的功能都會殺死這個賣點。

## 什麼留在 client-side（永遠）
- RAW 內嵌預覽抽取、銳利度評分
- 連拍/相似群組、閉眼/人臉偵測（mediapipe / face 模型在瀏覽器跑）
- 選片、評分、100% loupe、匯出選用清單、複製檔案、XMP 星等寫回
- DNG/CR3 全 RAW 解碼（LibRaw-wasm，仍在瀏覽器）

→ 結論：**整條照片管線在瀏覽器完成，zero backend。** 這也讓它免費、零伺服器成本、
部署到任何靜態主機。

## 後端只有在加「跟照片無關」的功能時才需要
若未來要做這些，才考慮 thin serverless（Cloudflare Workers / Supabase），
**且只碰 metadata、絕不碰圖**：
- 跨裝置帳號 / 設定同步
- 匿名使用統計
- （若哪天要變現）金流

## 判斷準則（加任何功能前自問）
> 這個功能需要把使用者的照片或像素送出瀏覽器嗎？
> 需要 → 不要做（或改成本機方案）。不需要 → 可以做，且優先純前端。
